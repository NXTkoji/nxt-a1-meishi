Design — Duplicate Review UI (Revised)

Same field-editor style as post-analysis, two records side by side

Concept

When a duplicate is detected, the card's review panel expands into a two-column field editor. Left = existing record. Right = new card. The user moves, adds, or deletes fields freely across both sides — just like the existing post-analysis editor. Whatever is in the left column when they confirm becomes the saved state of the existing person. The new card's image is always linked to the person regardless of choice.

Field highlighting

Fields highlighted in green on the right = new data not present in existing record (new phone, updated title, etc.). Same-value fields shown in normal style. This helps the user quickly spot what's actually different.
Does this direction look right? The → and ← arrows move a field from one column to the other. ✕ deletes it entirely. "Not a duplicate" dismisses the panel and saves the new card as a fresh person. Does this match what you had in mind?
