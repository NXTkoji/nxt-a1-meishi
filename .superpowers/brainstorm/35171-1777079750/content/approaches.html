3 Design Approaches

How should duplicate check and export integrate into the existing app?

A

Inline Extension Recommended

Duplicate panel expands inline within the existing review card. Export mode adds a checkbox overlay and floating action bar to the existing card list — no new screens.

Pros

  • No new navigation or screens to learn
  • Duplicate decision happens in context of the card image
  • Fastest to implement

Cons

  • Review card becomes tall/busy when duplicate panel is open
B

Dedicated Screens

Duplicate check becomes its own full-screen step between analysis and confirm. Export gets its own page accessible from a nav menu item.

Pros

  • Cleaner, focused UX for each task
  • Export page is easy to find

Cons

  • Adds a step to every scan even when no duplicate exists
  • More screens to build and maintain
C

Modal-First

Duplicate check pops a full-screen modal when a match is found. Export is a multi-step modal/wizard triggered from the card list. No new pages at all.

  • Modal fatigue if scanning many cards with duplicates
